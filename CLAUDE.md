## Project Overview

eBot-docker is a Docker Compose orchestration for the eBot CS2 match server ecosystem. It wires together:

- **[eBot-CS2-Web](https://github.com/iolson/eBot-CS2-Web)** — Laravel 12/PHP 8.4 web admin panel
- **[eBot-CSGO](https://github.com/deStrO/eBot-CSGO)** — Node.js + PHP match socket server (ebot-socket)
- **[eBot-LogReceiver](https://github.com/iolson/eBot-LogReceiver)** (branch: `io-modernizer`) — Node.js UDP log receiver
- MySQL 8.4, Redis 7, Nginx

## Initial Setup

```bash
cp .env.sample .env
chmod a+x setup.sh configure.sh
./setup.sh            # Interactive: configure .env with IPs, passwords, admin account
./configure.sh        # Generate etc/eBotWeb/.env and patch etc/eBotSocket/config.ini from .env
docker compose build
docker compose up
```

`configure.sh` must be re-run any time `.env` changes.

## Common Commands

```bash
docker compose up           # Start all services
docker compose up -d        # Start detached
docker compose build        # Rebuild images (re-clones repos from GitHub)
docker compose logs -f      # Follow all logs
docker compose logs -f ebot-web   # Follow a single service
docker compose down         # Stop and remove containers
docker compose down -v      # Also remove volumes (resets DB and app state — destructive)
```

## Architecture

### Services (docker-compose.yml)

| Service | Image/Context | Port | Role |
|---|---|---|---|
| `ebot-web` | `./ebot-web` (PHP 8.4-FPM) | internal | Laravel 12 web app (clones `iolson/eBot-CS2-Web`) |
| `ebot-socket` | `./ebot-socket` (PHP 7.4 + Node.js) | 12360 | Node.js socket + PHP bootstrap for match control |
| `ebot-logs-receiver` | `./ebot-logs-receiver` (Node 22) | 12345/udp | UDP log receiver (clones `iolson/eBot-LogReceiver@io-modernizer`) |
| `nginx` | `nginx:alpine` | 80, 443 | Reverse proxy — serves `eBotWebPublic` volume + proxies PHP to `ebot-web:9000` |
| `mysqldb` | `mysql:8.4` | internal | Database (shared between web and socket) |
| `redis` | `redis:7-alpine` | internal | Message queue between log receiver and socket |

### Named Volumes

| Volume | Purpose |
|---|---|
| `eBotWebStorage` | Laravel `storage/` directory (sessions, cache, queued jobs) |
| `eBotWebPublic` | Laravel `public/` with built Vite assets — shared between `ebot-web` and `nginx` |

### Configuration files (etc/)

These are mounted into containers at runtime. Run `./configure.sh` to regenerate from `.env`.

- `etc/eBotWeb/.env` — **Generated by configure.sh.** Laravel .env for the web container. Gitignored.
- `etc/eBotSocket/config.ini` — Socket server config (MySQL, Redis, websocket secret). Patched by configure.sh.
- `etc/eBotLogReceiver/config.json` — Log receiver config (UDP bind, Redis emitter).
- `etc/nginx/default.conf` — Nginx server block (Laravel routing, proxies to `ebot-web:9000`).

### Key .env variables (root .env)

| Variable | Purpose |
|---|---|
| `EBOT_IP` | Public/LAN IP of this server |
| `EBOT_WEBSOCKET_URL` | Full WebSocket URL for browser connection (e.g. `http://IP:12360` or SSL domain) |
| `EBOT_WEBSOCKET_SECRET_KEY` | Shared JWT secret between web and socket services |
| `LOG_ADDRESS_SERVER` | UDP address CS2 servers send logs to (e.g. `udp://IP:12345`) |
| `EBOT_ADMIN_*` | Initial admin account credentials (used on first container start) |
| `MYSQL_*` | Database credentials |

### Service startup (ebot-web)

`ebot-web/setup.sh` runs on container start:
- **First run** (no `storage/app/.ebot_installed`): generates Laravel app key, runs `php artisan migrate`, creates admin user from `EBOT_ADMIN_*` env vars, marks installed.
- **All runs**: caches config/routes/views, then starts php-fpm.

### Log receiver (ebot-logs-receiver)

Binds UDP on port 12345. CS2 game servers must be configured to send logs to `udp://HOST:12345`. Logs are published to Redis where ebot-socket subscribes.

### SSL Support

SSL termination is handled externally (e.g., Certbot + a separate reverse proxy). Set `EBOT_WEBSOCKET_URL` to your `wss://` or `https://` address and route traffic to port 12360 via your proxy.
